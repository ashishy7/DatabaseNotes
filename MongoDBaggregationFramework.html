  • Wraps the $count aggregation operator to produce the number of documents in a collection based on the collection’s metadata
  • The operation does not perform the find() query but instead counts the results that would be returned by the query
  • db.customers.count() returns a count of all customers
  • Counts the number of documents referenced by a cursor that is returned by find() query

  • Returns the count of all documents in a collection or view, does not take a query filter, and instead uses metadata to return the count for a collection
  • It should be used when there are a large number of documents
  • Wraps the distinct command to find distinct values for a specified field across a collection and return a document that contains an array of the distinct values

Using the Aggregation Pipeline (1 of 2)

The MongoDB aggregation pipeline framework consists of the aggregate() collection method, and a

sequence of operations referred to as stages. This sequence is referred to as a pipeline.

Documents enter a multi-stage pipeline that transforms the documents into aggregated results.

  • If you place a field in quotes, but precede it with a dollar sign (for example, $customer.name), the parser will return the value of the field.
