Using the Aggregation Pipeline (1 of 2)

The MongoDB aggregation pipeline framework consists of the aggregate() collection method, and a

sequence of operations referred to as stages. This sequence is referred to as a pipeline.

Documents enter a multi-stage pipeline that transforms the documents into aggregated results.

  • If you place a field in quotes, but precede it with a dollar sign (for example, $customer.name), the parser will return the value of the field.

Using the Aggregation Pipeline (2 of 2)

The MongoDB aggregation pipeline consists of stages. Each stage transforms the documents as they pass through the pipeline. Pipeline stages do not need to produce one output document for every input document; e.g., some stages

may generate new documents or filter out documents.

Pipeline stages can appear multiple times in the pipeline with the exception of $out,

Aggregation Pipeline Operators

Expression operators are are used to construct expressions

that can be used in the aggregation pipeline stages

Format: { <operator>: <argument> } or

{ <operator>: [ <argument1>, <argument2> ... ] }

Example:

db.sales.aggregate(

[ { $project: { item: 1, total: { $add: [ "$price", "$fee" ] } } } ] )

Expressions can include field paths and system

variables, literals, expression objects, and expression operators. Expressions can be nested.
