CRUD Operations

Operation Mongo Operator Notes
Create db.collection.insertOne() db.collection.insertMany() Insert operations target a single collection. All write operations in MongoDB are atomic on the level of a single document.
Read db.collection.find() Read operations retrieves documents from a collection
Update db.collection.updateOne() db.collection.updateMany() db.collection.replaceOne() Update operations modify existing documents in a collection, target a single collection, and are atomic on the level of a single document.
Delete db.collection.deleteOne() db.collection.deleteMany() Db.collection.remove() Delete operations remove documents from a collection, target a single collection, and are atomic on the level of a single document.

SQL Queries vs Mongo Queries

Assume SQL table people and Mongo collection called people with attributes: object id, user_id, age, status

SQL Term/Concept

CREATE TABLE people ( id MEDIUMINT NOT NULL AUTO_INCREMENT, user_id Varchar(30), age Number, status char(1), PRIMARY KEY (id) )

CREATE INDEX idx_user_id_asc ON people(user_id)

DROP TABLE people

SELECT * FROM people SELECT user_id, status FROM people WHERE status = "A SELECT * FROM people WHERE status = "A" OR age != 50 SELECT * FROM people WHERE user_id like "bc%"

SELECT * FROM people WHERE status = "A" ORDER BY user_id DESC

SELECT COUNT(*) FROM people

SELECT COUNT(user_id) FROM people

SELECT DISTINCT(status) FROM people

SELECT * FROM people LIMIT 5 SKIP 10

EXPLAIN SELECT * FROM people WHERE status = "A"

UPDATE people SET status = "C" WHERE age > 25

UPDATE people SET age = age + 3 WHERE status = "A"

Mongo Term/Concept

db.people.insertOne( { user_id: "abc123", age: 55, status: "A" } )

  • Table is implicitly created on first insertOne() or insertMany()
    • peration.
  • The primary key _id is automatically added if _id field is not specified.

db.people.createIndex( { user_id: 1, age: -1 } )

db.people.drop()

db.people.find() db.people.find( { status: "A" }, { user_id: 1, status: 1, _id: 0 } ) db.people.find( { $or: [ { status: "A" } , { age: {$ne: 50} } ] } ) db.people.find( { user_id: /^bc/ } ) -or db.people.find( { user_id: { $regex: /^bc/ } } )

db.people.find( { status: "A" } ).sort( { user_id: -1 } ) //descending db.people.find( { status: "A" } ).sort( { user_id: 1 } ) //ascending

db.people.count() or db.people.find().count()

db.people.count( { user_id: { $exists: true } } )

db.people.aggregate( [ { $group : { _id : "$status" } } ] )

db.people.find().limit(5).skip(10)

db.people.find( { status: "A" } ).explain()

db.people.updateMany( { age: { $gt: 25 } }, { $set: { status: "C" } } )

db.people.updateMany( { status: "A" } , { $inc: { age: 3 } } )

DELETE FROM people WHERE status = "D"

db.people.deleteMany( { status: "D" } )
