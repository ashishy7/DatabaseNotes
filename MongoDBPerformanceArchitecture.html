MongoDB Advantages

Characteristic Advantage
Document- oriented Data is stored in the database in a format that is very close to what you will be dealing with in both server-side and client-side scripts. This eliminates the need to transfer data from rows to objects and back
High performance BSON-based operations provide high performance.
High availability and scalability Its replication model provides high availaiblity. Sharding of data enables horizontal scalability
No SQL injection Not susceptible to SQL injection (putting SQL statements in web forms or other input from the browser that compromises the DB security) because objects are stored as objects, not by using SQL strings.

Flexible Storage Architecture

Auto-Sharding for DB Scalability (1 of 2)

  • You would normally configure a mongos (MongoDB shard routing service) instance to route reads and writes to specific shards in the cluster.
  • The mongos instances are lightweight and can greatly enhance performance and scalability, especially when dealing with massive amounts of data.
  • Config servers, which must be a replica set, are used to store metadata for a sharded cluster
